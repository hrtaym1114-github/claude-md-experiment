## 概要

Claude CodeはAnthropicが提供する公式CLIツールで、ターミナルから直接AIと対話しながら開発・執筆・管理作業を行える。本記事では、CLAUDE.mdの設計からカスタムコマンド、サブエージェント活用まで、実践的な活用方法を解説する。

## 本文

### 1. Claude Codeとは

Claude Codeは、ターミナル上でClaudeと対話しながらファイル操作、コード生成、タスク自動化ができるCLIツールだ。従来のチャットUIとは異なり、プロジェクトのコンテキストを直接理解し、ファイルの読み書きや検索を自律的に行える。

### 2. CLAUDE.mdでプロジェクトを定義する

Claude Codeの効果を最大化する鍵はCLAUDE.mdの設計にある。このファイルにプロジェクトの構造、命名規則、ワークフローを記述することで、AIがプロジェクト固有の文脈を理解して動作する。

記述すべき主要項目:
- フォルダ構成とその役割
- ファイル命名規則
- 利用可能なコマンド一覧
- 配置判断ルール
- 作業範囲の制限

### 3. カスタムコマンドで日常業務を自動化

`.claude/commands/` にMarkdownファイルを配置することで、カスタムスラッシュコマンドを定義できる。

活用例:
- `/daily` — デイリーノートを自動作成
- `/memo-to-inbox` — メモの自動分類・振り分け
- `/research [テーマ]` — テーマの網羅的リサーチ
- `/blog-generate [テーマ]` — ブログ記事の下書き生成

### 4. .claude/rules/で条件付きルールを分離

CLAUDE.mdが肥大化する場合、条件付きルールを `.claude/rules/` に分離できる。ファイルパスやコンテキストに基づいて自動適用されるため、CLAUDE.md本体をスリムに保てる。

### 5. サブエージェントで並列タスク実行

Claude CodeのAgent Toolを使えば、独立したタスクを複数のサブエージェントに分散して並列実行できる。ただし、同一ファイルへの同時書込みは禁止という排他制御が必要になる。

### 6. Hooksで自動ログ・振り返り

`beforeSubmitPrompt`、`afterFileEdit`、`stop` の各フックを活用して、セッションの自動ログ記録や振り返り生成が可能。

## まとめ

Claude Codeは単なるコード生成ツールではなく、プロジェクト全体のワークフローを自動化する基盤になりうる。CLAUDE.mdの設計品質が活用効果を左右するため、最初の設計に時間をかけることが重要だ。
