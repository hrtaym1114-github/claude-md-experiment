# ブログ記事ルール

## 対象
`05_Output/Areas/Content-Creation/Blog-Writing/` 配下の記事

## スラッグ（必須）

| ルール | 詳細 |
|--------|------|
| 言語 | 英語のみ（日本語スラッグ禁止） |
| 文字 | 小文字、ハイフン区切り |
| 長さ | 60文字以内 |
| 内容 | SEOキーワードを含める |
| 省略語 | the, a, an, of, for, and, in, on, at, to, is, are, was, with を省略 |
| 年号 | 時事性のある記事は末尾に `-2026` |

### 良い例
| タイトル | スラッグ |
|---------|---------|
| Claude Code完全ガイド | `claude-code-complete-guide-2026` |
| Whisper vs Moonshine比較 | `whisper-vs-moonshine-benchmark-2026` |

## メタデータ（必須フィールド）

```markdown
## メタデータ
- **ステータス**: 下書き
- **スラッグ**: [英語SEOスラッグ]
- **メタディスクリプション**: [120-160文字]
- **カテゴリ**: [AI活用/生産性/開発/レビュー]
- **タグ**: [カンマ区切り]
- **ファクトチェック**: 未
```

## SEO構成基準

| 項目 | 基準 |
|------|------|
| 文字数 | 2000文字以上（3000文字以上推奨） |
| 見出し | h2: 3-7個、h3: 必要に応じて |
| 導入 | 最初の200文字にキーワードを含める |
| 結論 | 必ずまとめセクションを入れる |

## ファクトチェック

| ステータス | 記載 |
|-----------|------|
| 未実施 | `- **ファクトチェック**: 未` |
| 実施済み | `- **ファクトチェック**: 済（YYYY-MM-DD ツール名）` |
