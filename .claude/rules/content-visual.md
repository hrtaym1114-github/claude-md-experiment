# コンテンツ図解ルール

## 図解候補の自動検出

記事本文をスキャンし、以下のパターンがある箇所を「図解候補」として検出する:

| パターン | 例 | 推奨図タイプ |
|---------|---|------------|
| 3ステップ以上の手順 | 「Step 1: ...」 | フローチャート |
| 3項目以上の比較 | 「A vs B vs C」 | 比較図 |
| システム構成の説明 | 「〜が〜に送信」 | アーキテクチャ図 |
| 数値・データの列挙 | 「30%改善」「3倍速い」 | グラフ |

## 検出後の処理

1. 図解候補が1箇所以上ある場合 → ユーザーに候補一覧を提示
2. 「図解を生成しますか？」と確認
3. 承認後に生成（ツールは状況に応じて選択）

## スキップ条件

- 記事が300文字未満
- ユーザーが `--no-visual` を指定
- 既に図解が存在する記事の再処理
