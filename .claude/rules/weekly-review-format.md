# 週次レビューフォーマット

## 保存先
`02_Daily/YYYY/YYYY-MM/YYYY-WNN-週次レビュー.md`

例: `02_Daily/2026/2026-03/2026-W12-週次レビュー.md`

## 出力テンプレート

```markdown
# YYYY-WNN 週次レビュー（MM/DD〜MM/DD）

## 📊 今週のサマリー
- **完了タスク**: N個
- **未完了タスク**: N個
- **作成ファイル**: N個
- **整理ファイル**: N個

## ✅ 完了したタスク
- [x] タスク1
- [x] タスク2

## ⬜ 未完了・持ち越し
- [ ] タスク1 → 来週に持ち越し
- [ ] タスク2 → 優先度見直し

## 📁 今週の活動
### 新規作成ファイル
- `パス/ファイル名` — [概要]

### 03_Input/ 新規追加
- `パス/ファイル名` — [概要]

## 🔍 振り返り
### うまくいったこと
-

### 改善すべきこと
-

### 気づき
-

## 🎯 来週のTop 3 Priorities
1.
2.
3.

## 📂 プロジェクト進捗
| プロジェクト | 状態 | 今週の進捗 | 来週の予定 |
|------------|------|-----------|-----------|

#weekly-review #YYYY-WNN #YYYY-MM
```

## 実行手順

1. `02_Daily/YYYY/YYYY-MM/` の今週分デイリーノートを全て読む
2. 各デイリーのタスク（チェックボックス）を集計
3. `03_Input/` の今週追加ファイルを確認
4. `05_Output/Projects/@Active/` の進捗を確認
5. テンプレートに沿って週次レビューを作成
6. 来週のTop 3 Prioritiesを提案

## データ不足時の対応

デイリーノートが少ない場合:
- 存在するデイリーのみから集計
- 「今週のデイリーノート: N/7日分」と明記
- git log やファイルの更新日時から活動を補完
