# CLAUDE.md

このファイルはClaude Codeがこのリポジトリで作業する際のガイダンスを提供します。

## セッション開始時の案内

ユーザーが挨拶（「おはよう」「hi」「hello」等）や曖昧な入力で開始した場合、以下を出力:

```
このVaultでは以下のことができます:

**日常運用**
- `/daily` — 今日のデイリーノート作成
- `/memo-to-inbox` — メモの整理・構造化

**コンテンツ制作**
- `/blog-generate [テーマ]` — ブログ記事下書き生成
- `/research [テーマ]` — テーマを網羅的に調査

**振り返り**
- `/weekly-review` — 週次レビュー実行

何をしましょうか？
```

**注意**: 具体的なタスク指示の場合はこの案内を出さず、直接タスクを実行する。

## 概要

PARA法とZettelkastenを組み合わせたObsidianベースの個人知識管理システム（セカンドブレイン）。

## フォルダ構成

```
00_Memo/          - クイックキャプチャ（生の入力）- 整理不要
01_Inbox/         - 処理済みファイル（タグ・リンク付き）
02_Daily/         - デイリーノート（YYYY/YYYY-MM/YYYY-MM-DD.md）
03_Input/         - 短期記憶（今週〜今月の資料）
04_Memory/        - 長期知識ストレージ（MOC付きでカテゴリ分類）
05_Output/        - プロジェクト（@Active/@Planning/@Completed）& エリア
06_Templates/     - 各種テンプレート
07_System/        - ダッシュボード、スクリプト、設定
99_Archive/       - 年別アーカイブ
```

## ファイル配置の判断フロー

```
新しいファイルの配置先を決める:
  1. とりあえずメモ → 00_Memo/
  2. 今週使う資料？ → 03_Input/[テーマ]/
  3. 3ヶ月後も参照する知識？ → 04_Memory/[カテゴリ]/
  4. 期限ありプロジェクト？ → 05_Output/Projects/
  5. 期限なし継続活動？ → 05_Output/Areas/
  6. 迷ったら → 01_Inbox/（週次レビューで処理）
```

### 03_Input と 04_Memory の判断基準

| 条件 | 配置先 | 例 |
|------|--------|---|
| 今のプロジェクトの素材 | 03_Input/ | リサーチ結果、記事素材 |
| 恒久的な参照知識 | 04_Memory/ | ツール設定、概念ノート |
| 素材 → 記事化完了後 | 04_Memory/ に移動 | Blog記事の元ネタ |

### 04_Memory カテゴリ判断基準

| カテゴリ | 内容 | キーワード例 |
|---------|------|------------|
| AI/ | AIツール、LLM、プロンプト | Claude, ChatGPT, Whisper, AI |
| Technical/ | プログラミング、インフラ、非AI技術 | Python, Docker, Git, Linux |
| Business/ | ビジネス戦略、コンテンツ戦略 | SEO, マーケティング, 収益化 |
| Personal/ | 個人の学習、キャリア | 学習, スキル, キャリア |

## メモ整理（/memo-to-inbox）のルール

### 振り分け先の判定

00_Memo/ のファイルを分析し、以下のスコアリングで配置先を決定:

**ブログネタスコア（0-10）**:
- 文字数: 2000字以上=3点, 1000-1999=2点, 500-999=1点, 300-499=0.5点, 300未満=0点
- 構造: 見出し5個以上+表リスト複数=3点, 見出し3-4個=2点, 見出し2個=1点
- 深度: 考察系キーワード（分析,比較,検証）=最大2点 + 実用系キーワード（手順,ガイド）=最大2点

**振り分け判定**:
- 7点以上 → `03_Input/Blog-Research/[テーマ]/` に直行
- 5-6点 → `03_Input/Blog-Research/[テーマ]/` に候補として配置
- 3点未満 → `01_Inbox/` に通常振り分け

### 元ファイルの扱い

振り分け後、元の00_Memo/ファイルは**そのまま残す**（削除禁止ルール）。新しい場所にコピーとして作成する。

## 作業範囲の制限

- このVault内でのみ作業
- ファイル削除禁止（rm, unlink等）
- フォルダ削除禁止
- 許可: 読み取り、新規作成、編集、移動、検索、フォルダ作成

## 基本原則

1. **完璧主義を捨てる** — まず記録、整理は後から
2. **情報は流れる** — 00_Memo → 01_Inbox → 適切な場所 → 99_Archive
3. **リンクが知識を作る** — 価値は繋がりから、フォルダ階層ではない

## AI支援のガイダンス

- ファイルが特定の場所に属する**理由**を常に説明する
- 改善は優しく提案、ルールを強制しない
- **曜日は必ず `date` コマンドで確認してから言及する**（推測禁止）
- 完璧より継続性を優先
